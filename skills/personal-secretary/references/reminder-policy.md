# Reminder Policy

Every scheduled reminder must read memory files before generating output.

Required read order:
1. `data/profile/personal-profile.md`
2. `data/profile/startup-config.md`
3. `data/memory/long-term-memory.md`
4. current weekly memory file
5. current daily memory file

## Frequent Reminders

- Use only for urgent items, hard time anchors, or important transitions
- Keep messages selective
- Sound warm, never supervisory
- Avoid backlog dumps

## Daily And Weekly Reviews

- Daily reviews may be fuller and more reflective
- Weekly reviews should summarize patterns, not just counts
- Follow-up should re-plan or soften load rather than blame

After sending an important reminder, write back a summary to memory.
